## 2. ストレステスト（6シナリオ）

### シナリオ1: GoogleがpSEOサイトを全面禁止

**前提**: Google March 2026 Core Updateの延長線上で、テンプレートベースの自動生成ページがインデックスから完全排除される。sushihey.comのSEO流入が永久にゼロ。

**影響を受ける収益源**:
- 広告収入（SEO経由トラフィック）: **全滅**
- アフィリエイト収入（SEO経由）: **全滅**

**影響を受けない収益源**:
- YouTube広告収入
- SNS経由の直接トラフィック
- Pinterest経由のトラフィック
- Newsletter購読者からの直接訪問
- キャラクターIP（グッズ、スタンプ、ライセンス）

**定量分析**:

Ag2の設計では、12ヶ月目にSEOトラフィックが月間50,000セッション→Raptive/Mediavineで月$1,000-$1,500の広告収入を想定。これが消える。

代替チャネルの収益ポテンシャル:

| チャネル | 12ヶ月目の想定月間訪問 | 広告RPM | 月間収益 | エビデンス |
|---------|---------------------|---------|---------|-----------|
| Pinterest→サイト | 5,000-15,000 | $10-15 | $50-225 | フードPinのCTR 1.0-1.6%（WebFX） |
| YouTube説明欄→サイト | 2,000-5,000 | $10-15 | $20-75 | 教育系CTR 2-5%（VidIQ） |
| Newsletter→サイト | 1,000-3,000 | $10-15 | $10-45 | 開封率20-30%×クリック率3-5% |
| 直接/ブックマーク | 500-1,000 | $10-15 | $5-15 | ブランド認知による |
| **合計** | **8,500-24,000** | | **$85-360** | |

**結論**: SEOゼロでも月$85-360の広告収入は残る。ただし**Mediavine Journey（1,000セッション/月）の閾値は超えられる**（Productive Blogging）ため、広告ネットワーク自体は使える。しかし、Raptive（25,000PV/月）には届かない可能性が高い。

**SUSHIHEY全体として成立するか？**:

| 収益源 | SEOゼロでの月間収益（12ヶ月目） |
|--------|-----|
| サイト広告（非SEOトラフィック） | $85-360 |
| YouTube広告（1万登録・英語圏CPM $5-15） | $200-600 |
| キャラクターグッズ（SUZURI等） | $0-50 |
| LINEスタンプ | $0-3 |
| **合計** | **$285-1,013** |
| **円換算（1$=150円）** | **42,750-151,950円** |

**判定**: SEO全滅でも月4-15万円は残る。**成立はするが、当初の目標（月30万円以上）には大幅に届かない**。

**対策（弱点→対策→対策の弱点のサイクル）**:

- **対策1**: YouTube/Pinterestの直接収益化に軸足を移す
  - 弱点: YouTube収益化まで15.5ヶ月（VidIQ平均）。その間の収入がゼロ
  - 対策1-1: Shorts 1,000万再生ルート（90日プラン、VidIQ）を採用し、広告収入の前にShortsファンドで$100-500/月を狙う
  - 弱点1-1: Shorts再生のみでは広告単価が低い（Shorts RPM $0.04-0.06、Shorts Fund RPM推定$0.03-0.07）

- **対策2**: Newsletter（Substack）を主要チャネルに昇格
  - 弱点: Newsletter購読者の獲得にSNSフォロワーが必要（鶏と卵問題）
  - 対策2-1: Reddit r/sushi（45.2万人）での信頼構築→Newsletter誘導
  - 弱点2-1: Redditは自己宣伝に厳しい（投稿の10%以下、数週間〜数ヶ月の純粋参加が必要）

### シナリオ2: YouTube/TikTokがAI生成動画を全面禁止

**前提**: 2025年7月のYouTube「inauthentic content」ポリシーがさらに厳格化。AI生成映像・音声を含むすべての動画が収益化不可に。TikTokも同様の措置。

**現実性の評価**:
- **低〜中**。2026年1月時点で100万以上のチャンネルがYouTubeのAIツールを使用（The Wrap）。完全禁止はYouTubeの収益にも打撃。
- ただし、AI「のみ」の動画は既に厳しい。2026年1月の大規模執行で16チャンネル永久停止、合計3,500万登録者が消失（Flocker）。

**影響を受ける収益源**:
- YouTube広告収入（AI動画分）: **全滅**
- TikTok Creator Rewards: **全滅**
- IP動画によるブランド認知: **大幅低下**

**影響を受けない収益源**:
- サイトSEO収入
- Pinterest（静止画ベース）
- Reddit/Newsletter
- キャラクターグッズ

**SUSHIHEY全体の収益構造（動画なし）**:

| 収益源 | 動画なしでの月間収益（12ヶ月目） |
|--------|-----|
| サイト広告（SEO） | $500-1,500 |
| アフィリエイト（Booking.com, Amazon） | $50-200 |
| Pinterest→サイト | $30-100 |
| キャラクターグッズ（静止画ベース） | $0-50 |
| **合計** | **$580-1,850** |
| **円換算** | **87,000-277,500円** |

**判定**: **記事＋データだけでもSUSHIHEYは成立する**。SEO戦略がAg2の設計通りに機能すれば、動画なしでも月9-28万円。ただし、キャラクターIPの育成速度は大幅に低下する。

**対策**:
- **対策1**: 人間の関与を最大化した動画に移行（自分の声でナレーション、手描きのフレーム挿入、人間の監修を明示）
  - 弱点: 松宮さんの時間が増大する
  - 対策1-1: テンプレート化した制作フロー（AI下書き→人間が修正→AIで仕上げ→人間が最終確認）

- **対策2**: 静止画コンテンツ（インフォグラフィック、マンガ形式）に軸足を移す
  - 弱点: 動画ほどのバイラル力がない
  - 対策2-1: Pinterestに集中。フード系セーブ率は1-2%でプラットフォーム平均の2倍（WebFX）。ピンの寿命3.88ヶ月はSNS中最長。

### シナリオ3: 大手メディアが英語圏の寿司権威サイトを立ち上げ

**前提**: Eater, Serious Eats, Bon Appetitのいずれかが「sushi.eater.com」のような寿司特化サブドメインを立ち上げ、DA 80以上のドメインパワーで攻めてくる。

**現実性の評価**:
- **中**。現時点で英語の寿司専門メディアは空白地帯（The Sushi Geek程度）。大手は寿司を「カテゴリの一つ」としか扱っていない。
- しかし、SUSHIHEYが成功すれば追随者が出る可能性は高い。

**SUSHIHEYの差別化要素と防御壁**:

| 差別化要素 | 大手が真似しにくい理由 | 防御力 |
|-----------|---------------------|--------|
| **SUSHI HEY Score（5ソース合成）** | 食べログ+Google+ミシュラン+OAD+百名店のデータ合成は日本在住者の優位性 | ★★★★☆ |
| **200種の寿司ネタ図鑑（味覚プロファイル付き）** | 規模の壁。一から200種を作るのは大手でも6ヶ月以上 | ★★★★☆ |
| **旬データの月次自動更新** | 豊洲市場データのcronジョブ（VPS既存）。リアルタイム性で差別化 | ★★★★★ |
| **キャラクターIP（小肌くん他）** | 大手メディアはキャラクターIPを持たない。コンテンツの差別化として機能 | ★★★★★ |
| **コミュニティ（英語UGCレビュー）** | ネットワーク効果。先行者がレビューを蓄積するほど後発が不利 | ★★★☆☆（初期は脆弱） |
| **日本在住×ネイティブレベル英語** | EaterはNYC拠点。東京の寿司店を毎週訪問して一次情報を書ける人材は希少 | ★★★★★ |

**先行者利益の持続期間**:

- **SEO面**: 新規ドメインが有意なトラフィックを得るまで6-12ヶ月（Ahrefs）。大手がサブドメインで攻める場合でも、メインドメインのDAが効くまで3-6ヶ月。SUSHIHEYが12ヶ月先行していれば、データ量・内部リンク構造・UGCで優位を保てる。
- **IP面**: キャラクターIPは時間経過とともに認知が複利で積み上がる。大手が後から同様のキャラを作っても、ファンベースの蓄積は追い越せない。
- **データ面**: SUSHI HEY Scoreの精度は時間とともに向上（データソースの増加、UGCの蓄積）。後発はDay 1のスコアしか出せない。

**判定**: **12ヶ月の先行で十分な防御壁を構築可能**。ただし、コミュニティ（UGC）の構築が遅れると、大手のDAパワーに押しつぶされるリスクあり。

**対策**:
- **対策1**: 最初の6ヶ月でUGCレビューを100件以上集める（Reddit r/sushi誘導＋メール依頼）
  - 弱点: 英語UGCの品質管理が困難
  - 対策1-1: AI+人間のモデレーション二重構造。不正レビュー検知を初期から設計（TripAdvisorモデル: 8.8%検知率）

### シナリオ4: 食べログがスクレイピングを完全ブロック

**前提**: 食べログが技術的・法的にスクレイピングを完全に防止。SUSHI HEY Scoreの核心データソース（35%の重み）が消失。

**現実性の評価**:
- **高**。食べログ利用規約は「営業活動目的のアクセス」を禁止（第6条(1)）。「複製、送付、譲渡、頒布」も禁止（第9条(1)）。法的措置の前例あり（韓流村事件は別の論点だが、データ利用への姿勢は厳格）。

**代替データソース**:

| データソース | 取得難易度 | コスト | カバー範囲 | SUSHI HEY Scoreへの組込可能性 |
|------------|-----------|-------|-----------|---------------------------|
| **Google Places API** | 低 | Essentials: 10,000件/月無料、超過$5/1000件 | 評点+レビュー数+写真+営業時間 | ★★★★☆ W3の0.15→0.35に引上げ可能 |
| **ミシュランガイド公式** | 中 | 無料（公開情報） | 星付き+ビブグルマン+プレート | ★★★★★ 既に組込済（W2=0.25） |
| **OADランキング** | 低 | 無料（公開情報） | トップ100店舗 | ★★★☆☆ 既に組込済（W4=0.10） |
| **TripAdvisor** | 中 | TripAdvisor Content API（申請制） | 英語圏レビュー | ★★★★☆ 外国人視点の代替 |
| **自社UGCレビュー** | 高（構築に時間） | 0円 | 段階的に拡大 | ★★★★★ 長期的には最強 |
| **Instagram API** | 中 | Meta Graph API（無料） | ハッシュタグ投稿数、写真 | ★★☆☆☆ 定性データのみ |
| **Googleマイビジネス（直接）** | 低 | 無料 | 営業情報、レビュー | ★★★★☆ Google Placesと重複 |

**修正版SUSHI HEY Score（食べログなし）**:

```
SUSHI_HEY_SCORE_v2 = W1×S_google + W2×S_michelin + W3×S_tripadvisor
                   + W4×S_oad + W5×S_hyakumeiten + W6×S_ugc
                   + TREND_BONUS + CONFIDENCE_PENALTY

重み:
W1 (Google Places)  = 0.30  ← 食べログの代替（外国人視点＋日本人レビューも含む）
W2 (ミシュラン)      = 0.25  ← 変更なし
W3 (TripAdvisor)    = 0.15  ← 新規追加（英語圏の権威）
W4 (OAD)            = 0.10  ← 変更なし
W5 (百名店)         = 0.05  ← 変更なし（公開リストとして利用）
W6 (自社UGC)        = 0.15  ← 新規追加（段階的に重み増加）
```

**判定**: **食べログなしでもSUSHI HEY Scoreは成立する**。ただし、日本人の「本場の評価」というE-E-ATシグナルが弱まる。対策として自社UGCの早期構築が不可欠。

**対策**:
- **対策1**: Google Places APIに軸足を移す（月10,000件無料、コスト$0）
  - 弱点: Googleの日本レストラン評点は食べログほど信頼されていない
  - 対策1-1: 複数ソースの乖離分析を「合成インサイト」として記事化（食べログがなくても、Google×ミシュラン×TripAdvisorの3ソース乖離で十分なユニークコンテンツが生成可能）

### シナリオ5: 松宮さんの時間がゼロ（月1時間のみ）

**前提**: 本業が繁忙期に入り、副業に割ける時間が月1時間以下。

**自動化で回る部分 vs 人間が必要な部分**:

| 作業 | 自動化可能性 | 月1時間で回せるか |
|------|------------|----------------|
| **豊洲市場データ取得（cron）** | ★★★★★ 完全自動 | ✅ 0分 |
| **旬ページの月次更新** | ★★★★☆ cronで自動生成→人間確認5分 | ✅ 5分 |
| **Google Places APIデータ同期** | ★★★★★ 完全自動 | ✅ 0分 |
| **SUSHI HEY Score計算** | ★★★★★ 完全自動 | ✅ 0分 |
| **AI動画生成** | ★★★☆☆ AI生成→人間がOK/NGを判断 | ❌ 1本あたり最低30分 |
| **SNS投稿** | ★★★★☆ Buffer/Laterで予約投稿可能 | ✅ 月1回バッチで設定（30分） |
| **Reddit投稿** | ★★☆☆☆ 自然な投稿は人間が必要 | ❌ 自動化するとバン |
| **UGCモデレーション** | ★★★★☆ AI自動+エスカレーション | ✅ 月2-3件の確認（5分） |
| **新規記事の執筆/編集** | ★★☆☆☆ AI下書き→人間編集が必須 | ❌ 1記事30分-1時間 |
| **コラボ/PR対応** | ★☆☆☆☆ 完全に人間が必要 | ❌ |

**月1時間で維持できる最低限の運用**:

1. 旬データの月次更新確認: 5分
2. SNS予約投稿のバッチ設定: 30分
3. UGCモデレーション確認: 5分
4. 月次KPIの確認: 10分
5. 緊急対応（あれば）: 10分

**月1時間運用での収益見込み（12ヶ月目）**:

| 収益源 | 月間収益 | 前提 |
|--------|--------|------|
| サイト広告（自動更新ページのみ） | $200-500 | 既存SEOトラフィックの維持（新規記事なし→トラフィック緩やかに減少） |
| アフィリエイト | $20-80 | 既存ページからの自動発生 |
| YouTube（既存動画のロングテール再生） | $50-150 | 新規動画なし→月次再生数は減少するが完全にはゼロにならない |
| **合計** | **$270-730** |
| **円換算** | **40,500-109,500円** |

**判定**: **月1時間でも月4-11万円は維持可能**。ただし成長は完全に停止する。これは「投資回収フェーズ」であり、「成長フェーズ」ではない。

**重要な前提**: この数字は「最初の6-12ヶ月で十分なコンテンツとSEOトラフィックを構築した後」の維持コスト。初期構築期に月1時間では何も始まらない。

**対策**:
- **対策1**: n8nワークフローで最大限自動化（VPS既存のn8n環境を活用）
  - AI記事の自動下書き→Slack通知→スマホで確認→1タップで公開
  - 弱点: 品質管理が甘くなるとGoogle Scaled Content Abuseに引っかかるリスク

- **対策2**: 繁忙期に入る前に3ヶ月分のコンテンツをバッチ制作
  - 弱点: 旬データの鮮度が落ちる
  - 対策2-1: 旬ページはcronで自動更新されるため鮮度問題なし。問題は「新規の人間的コンテンツ」が止まること。

### シナリオ6: sushihey.comのDNS問題が解決しない

**前提**: Cloudflareの管理者が見つからず、DNSレコードを変更できない。sushihey.comは使用不可。

**選択肢**:

| 選択肢 | メリット | デメリット | コスト |

## 5. 「複利度」による全施策の優先順位

### 5-1. 複利度の定義

**複利度** = 「一度実行したら、やるたびに効果が積み上がり、過去の成果が未来の成果を加速する度合い」

- **複利度5（最高）**: やればやるほど加速的に効果が増大。止めても価値が残る
- **複利度4**: 効果が積み上がるが、線形的。メンテナンスが必要
- **複利度3**: 一定の蓄積効果あり。ただし減衰もある
- **複利度2**: 限定的な蓄積。市場やプラットフォームの制約あり

## 6. 統合設計のメタ弱点分析

本設計書自体の弱点を自己批判し、対策を考え、さらにその対策の弱点を探る。

### 6-1. 弱点→対策→対策の弱点（3ラウンド）

#### ラウンド1

**弱点1**: 本設計はPV成長率を楽観的に見積もっている。12月に45,000-70,000PVは新規ドメインでは上位10%の成長。

- **対策1**: 悲観シナリオ（15,000-20,000PV、月収¥25,000-40,000）を正式な計画として採用し、楽観シナリオはボーナスとして扱う
- **対策1の弱点**: 悲観シナリオでは9ヶ月間の累計収入が¥100,000-150,000。投入時間を月80時間と仮定すると時給は¥139-208。モチベーション維持が困難
- **対策1-1**: 悲観シナリオでも「資産が積み上がっている」という事実を可視化するダッシュボードを構築（累計ページ数、DA推移、登録者数、フォロワー数）

**弱点2**: 全収益チャネルがプラットフォーム依存。Google、YouTube、Pinterestのいずれかがルール変更したら大打撃。

- **対策2**: 最も依存度の低い資産（キャラクターIP、UGCコミュニティ、Newsletter購読者リスト）に重点投資。これらは特定プラットフォームに依存しない
- **対策2の弱点**: キャラクターIPの収益化には1-3年かかる。その間のキャッシュフローが持たない
- **対策2-1**: Ag1のセルフバック＋メルカリで初月の生活費を確保。5-9月の「死の谷」は本業の給与で補填する前提とする

**弱点3**: 「英語圏の寿司専門メディアは空白地帯」という前提が、9ヶ月後も正しい保証がない。

- **対策3**: 6ヶ月目に競合調査を再実施。新規参入があれば差別化戦略を即時修正
- **対策3の弱点**: 6ヶ月後に気づいても、大手のDAパワーに追いつかれている可能性
- **対策3-1**: 参入障壁の高い資産（UGCレビュー、独自データ、キャラクターIP）の構築を最初の6ヶ月で最大化する

#### ラウンド2

**弱点4**: 松宮さんのスキルセット（英語力、デザイン力、動画制作力）が不明。全設計がAI自動化を前提としているが、品質管理は人間が必要。

- **対策4**: 最初の1週間で「パイロット版」を制作（1動画、1記事、1ピン）し、品質と制作時間を実測してからタイムラインを修正
- **対策4の弱点**: パイロット版の品質が低い場合、計画の大幅見直しが必要
- **対策4-1**: パイロット版が失敗しても、サイト構築（pSEO）は技術的に自動化できるため最低限の収益ラインは確保可能。動画は品質が安定してから開始

**弱点5**: 食べログスクレイピングの法的リスクを過小評価している可能性。

- **対策5**: 食べログのデータを「直接引用」ではなく「複数ソースの合成インサイト」として使用。食べログの評点そのものではなく、「5ソースの加重平均であるSUSHI HEY Score」として独自の価値を付加
- **対策5の弱点**: 「合成」であっても元データを取得する行為自体が利用規約に抵触する
- **対策5-1**: シナリオ4の通り、食べログを完全に排除したSUSHI HEY Score v2で運用する。Google Places API + ミシュラン + TripAdvisor + OAD + 自社UGCの5ソースで代替可能

#### ラウンド3

**弱点6**: 本設計書は「松宮さん1人 + AI」の体制を前提としている。ちいかわ（1人→60人チーム）、Kurzgesagt（1人→60人チーム）のような成功事例は、どこかのタイミングで「人を雇う」判断が必要。

- **対策6**: Year 1は1人+AI。月収が安定して¥100,000を超えたら、ライターやデザイナーを業務委託で1人追加。CW（クラウドワークス）で英語ライター月¥30,000程度
- **対策6の弱点**: 外注管理は時間がかかる。品質が不安定。ブランドの一貫性が損なわれるリスク
- **対策6-1**: 外注はpSEOの店舗ページデータ入力のみに限定。ブランドの声（Tone of Voice）が必要なコンテンツ（動画台本、SNS投稿、キャラクターの発言）は松宮さん+AIで維持

**弱点7**: 本設計は「日本在住」を前提としているが、松宮さんの居住地が不明。地方在住だとタイミー案件不足、豊洲市場訪問が困難等の問題。

- **対策7**: オンライン完結型の設計を基本とし、現地取材は「あればベター」の位置づけにする
- **対策7の弱点**: E-E-ATの「Experience（実体験）」シグナルが弱くなる
- **対策7-1**: 東京在住でなくても、年1-2回の「寿司巡り取材旅行」で十分な写真・体験を蓄積可能。これをコンテンツ化すれば一石二鳥
