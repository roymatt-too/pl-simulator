### プロンプト1: JD（職務記述書）自動生成

**用途**: クライアントからのヒアリング情報をもとに、求職者に響くJDを一発で生成する。

**プロンプト本文**:

```
あなたはプロの人材紹介コンサルタントです。以下の情報をもとに、求職者が「応募したい」と感じる魅力的なJD（職務記述書）を作成してください。

【入力情報】
- 企業名: {企業名}
- 業界: {業界}
- 職種: {職種名}
- ポジションの背景: {背景}
- 主な業務内容: {箇条書きで3〜5つ}
- 必須スキル・経験: {箇条書き}
- 歓迎スキル・経験: {箇条書き}
- 給与レンジ: {年収○○万〜○○万}
- 勤務地: {勤務地}
- リモートワーク: {可/不可/ハイブリッド}
- 企業の魅力・特徴: {箇条書きで2〜3つ}

【出力フォーマット】
1. 求人タイトル（30文字以内、魅力的なもの）
2. 企業紹介（100〜150文字）
3. 募集背景（50〜100文字）
4. 業務内容（箇条書き、具体的な動詞で始める）
5. 必須要件
6. 歓迎要件
7. 給与・待遇
8. 働き方・福利厚生
9. 選考プロセス
10. この求人の魅力ポイント（3つ）

トーンは「プロフェッショナルだが親しみやすい」で統一してください。
```

**使用例**:
- 入力: 企業名「スタートアップX社」、職種「バックエンドエンジニア」、年収600〜900万、フルリモート可
- 出力: 「【フルリモート】急成長SaaS企業のバックエンドエンジニア｜Go/Kubernetes環境」というタイトルに始まり、全セクション完備のJDが生成される

**カスタマイズTips**: 「トーンは〜」の部分を「カジュアル」「フォーマル」「ベンチャー感のある」などに差し替えると、企業カルチャーに合わせた文体に変わります。

### プロンプト2: 業界別の訴求ポイント提案

**用途**: 特定業界の求人で候補者に刺さる訴求ポイントをリストアップし、JDやスカウト文に活用する。

**プロンプト本文**:

```
あなたは人材紹介業界で10年以上の経験を持つシニアコンサルタントです。以下の業界・職種の求人について、求職者が最も魅力を感じる訴求ポイントを10個提案してください。

【条件】
- 業界: {業界名}
- 職種: {職種名}
- ターゲット候補者層: {例: 20代後半〜30代前半、現職は大手SIer}
- 企業の強み: {箇条書きで2〜3つ}

【出力要件】
各訴求ポイントについて:
1. 訴求ポイント（1行で）
2. なぜこのターゲットに響くのか（1〜2文）
3. JD/スカウト文での具体的な表現例（1文）

優先度が高い順に並べてください。
```

**使用例**:
- 入力: 業界「SaaS」、職種「カスタマーサクセス」、ターゲット「現職は人材業界の法人営業、28〜33歳」
- 出力: 「1. SaaSビジネスのスキルが身につく → 表現例:『営業力×テクノロジーのキャリアを実現』」のように10項目

**カスタマイズTips**: ターゲット候補者層をより具体的に書くほど、ピンポイントで刺さる訴求が出ます。

### プロンプト3: 給与レンジの市場相場分析

**用途**: 特定職種・業界の給与相場感を整理し、クライアントへの年収レンジ提案やJD記載に活用する。

**プロンプト本文**:

```
あなたは日本の転職市場に精通した報酬コンサルタントです。以下の条件に該当するポジションの給与相場を分析してください。

【条件】
- 職種: {職種名}
- 業界: {業界}
- 経験年数: {○年〜○年}
- 勤務地: {東京/大阪/地方都市/フルリモート}

【出力フォーマット】
1. 市場相場レンジ（年収: 下限〜上限）
2. 中央値
3. 上位25%ライン
4. 相場に影響する変動要因（3〜5つ）
5. 直近1年のトレンド
6. クライアントへの提案コメント例（50〜100文字）
```

**使用例**:
- 入力: 職種「Webアプリケーションエンジニア」、業界「SaaS」、経験3〜5年、東京、スタートアップ

**カスタマイズTips**: 「外資系と日系の比較表も出してください」を末尾に追加すると外資案件との比較提案にも使えます。

**用途**: クライアントの「あれもこれも必須」を合理的に仕分けし、応募ハードルを下げて母集団を増やす。

**プロンプト本文**:

```
あなたは人材紹介のシニアRAです。クライアントから以下のスキル・経験要件を預かりました。これらを「必須（MUST）」と「歓迎（WANT）」に仕分けしてください。

【クライアントが挙げた要件（全て）】
{要件を箇条書きで全て列挙}

【ポジション情報】
- 職種: {職種名}
- 年収レンジ: {年収}
- 想定される採用難易度: {高/中/低}

【出力フォーマット】
■ 必須（MUST）— これがないと業務遂行が不可能な要件のみ
- 要件1: 必須とする理由

■ 歓迎（WANT）— あれば活躍が加速するが、入社後の育成で補える要件
- 要件1: 歓迎に下げる理由

■ 削除推奨 — 年収レンジや採用難易度に照らして現実的でない要件
- 要件1: 削除推奨の理由

■ クライアントへの説明コメント（50〜100文字）
```

**使用例**:
- 入力: 職種「経理マネージャー」、要件「簿記1級、上場企業経験、IFRS、英語、マネジメント5年、SAP、M&A経験」、年収700〜900万
- 出力: MUST「簿記1級 or 同等実務、上場企業経験、マネジメント3年以上」→ WANT「IFRS、英語、SAP」→ 削除推奨「M&A経験（年収900万では採用困難）」

**カスタマイズTips**: 「採用競合として○○社と○○社がいます」を追加すると、競合の条件との比較で仕分けの精度が上がります。

### プロンプト5: 競合求人との差別化ポイント抽出

**用途**: 自社クライアントの求人を競合求人と比較し、差別化できるポイントを明確にする。

**プロンプト本文**:

```
あなたは転職市場分析のプロです。以下の2つの求人を比較し、【自社求人】の差別化ポイントを抽出してください。

【自社求人（クライアント）】
- 企業名: {企業名}
- 職種: {職種}
- 年収: {年収}
- 主な業務: {箇条書き}
- 働き方: {リモート/出社/ハイブリッド}
- その他特徴: {箇条書き}

【競合求人】
- 企業名: {競合企業名}
- 職種/年収/業務/働き方/特徴: {同様に記載}

【出力フォーマット】
1. 比較表（項目: 年収、働き方、成長性、福利厚生、技術スタック、カルチャー）
2. 自社求人が勝っている点（3つ）とその訴求文
3. 自社求人が劣っている点（3つ）とその補い方
4. 候補者への推薦時のトークスクリプト（100文字程度）
```

**使用例**:
- 入力: 自社「メガベンチャーA社、PM、800万」vs 競合「外資コンサルB社、PM、1000万」
- 出力: 「勝ち: 裁量権、WLB、SO / 劣り: 年収差200万 → 補い方: SOの期待リターンと昇給スピードで3年後の総報酬逆転を提示」

**カスタマイズTips**: 競合求人は2〜3社分入れると、市場全体でのポジショニングが見えます。

### プロンプト50: 年間目標設定と四半期ブレイクダウン

【前年実績】: 年間売上/成約数/平均単価/売上構成
